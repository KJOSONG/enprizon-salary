# 2026-06-04 工作日志

## 终端输出过多问题排查与解释
- 用户反馈 `./start.sh start` 启动后终端输出信息过多
- 结论：**所有输出均为正常行为，无任何错误**
- 输出分为四类：
  1. **启动横幅**：`app.py` 打印的系统信息（ENPRIZON LINDI PROJECT、130名员工、应发 52,291,805 TZS）
  2. **Flask 开发服务器警告**：标准提示"不要用于生产环境"，开发用可忽略
  3. **HTTP 访问日志（Werkzeug）**：浏览器加载页面时并发请求 10+ 个 API 端点（/salary、/employees、/production、/config、/audit-log 等），全部返回 200，系统正常运行
  4. **两个 404**：`/investment-dashboard.html`（其他标签页乱入）、`/favicon.ico`（无图标文件），均不影响功能
- 解决方案：如需减少终端日志，使用 `./start.sh bg` 后台启动，日志写入 `.kilwa.log`
