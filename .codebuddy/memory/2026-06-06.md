**效果**: 60万月薪/30天月 = 2万/天均匀分布；有A/L请假时总额与薪资总表完全对齐。

## 新增功能：Headless模式 + 员工离职管理 (2026-06-06 晚)

### 需求背景
- 可在源数据到达前切换到未来月份（如6月），提前手动记录出勤
- 手动出勤数据不应被后续上传源数据覆盖（已天然满足——attendance_overrides 独立于 Excel）
- 员工可标记离职并隐藏，工资表导出存档后从活跃列表移除

### 改动文件 (4个)

#### 1. `core/database.py` — dismissed_employees 表
- 新增 `dismissed_employees` 表 (employee_id PK, dismissed_at TEXT, note TEXT)
- 新增 `load_dismissed()` / `dismiss_employee()` / `restore_employee()` / `load_dismissed_with_info()` 四函数

#### 2. `app.py` — Headless模式 + 离职API
- **`APP_STATE`**: 新增 `headless: False` 字段
- **`get_available_months()`**: 扩展月份来源 → 当前月+未来2月 + attendance_overrides 表 + bonus_penalties 表
- **`_run_pipeline()`**: 硬排除之后增加离职员工过滤 (`load_dismissed`)
- **`set_month()`**: 检测到 dates 为空但员工存在时 → `calendar.monthrange` 生成当月全部日期 → 标记 `headless=True`；返回值包含 `headless` 字段
- **`get_attendance()` / `export_attendance()` / `export/all`**: `all_dates` 增加 `md.get('dates', [])` 覆盖生成日期
- **`GET /employees/dismissed`**: 获取已离职员工列表（含姓名）
- **`POST /employees/dismiss`**: 标记员工离职（写入DB + 从内存移除）
- **`POST /employees/restore`**: 恢复离职员工（重新加载管线）
- **`/employees` / `/salary`**: 返回值包含 `headless` 状态

#### 3. `templates/index.html` — 前端交互
- **数据台**: `#headlessBannerDash` 横幅（预览模式提示）
- **出勤页**: `#headlessBannerAtt` 横幅
- **员工表**: 每行新增 🗑 离职按钮 → `dismissEmployee()` 函数
- **筛选栏**: `#toggleDismissedBtn` 切换按钮 → `toggleDismissedList()` → 展开/收起 `#dismissedSection` 已离职列表
- **已离职列表**: 显示姓名、备注、恢复按钮（调用 `restoreEmployee()`）
- **`changeMonth()`**: 从响应读取 `data.headless` 并更新 `STATE._headless`
- **`loadAttendance()` / `renderDashboard()`**: 根据 `STATE._headless` 显示/隐藏横幅
- **审计标签**: 新增 `dismiss_employee` / `restore_employee`

#### 4. `static/js/i18n.js` — 翻译
- 离职相关: `emp_dismiss`, `emp_restore`, `emp_dismiss_confirm`, `emp_show_dismissed`, `emp_hide_dismissed`, `emp_dismissed_title`, `emp_no_dismissed`

## 产量图表月份切换残留修复 + 移除复位按钮 (2026-06-06 晚)

**问题1**: 切换到无数据的月份（如6月headless），产量图表canvas上仍显示旧月份（5月）数据。
**根因**: `renderProductionCharts` 在数据为空时跳过渲染，但未销毁旧 Chart.js 实例，canvas 残留旧图。

**修改 (templates/index.html)**:
- `renderProductionCharts`: 每个 `if` 增加 `else` 分支调用 `destroyChart(id)` 销毁旧图表
  - `chartDailyProd` 空 → 同时销毁 `chartProdPie`（pie图依赖同一数据源）
  - `chartDrillerProd` 空 → 销毁自身
  - `chartConsumables` 空 → 销毁自身
- 新增 `destroyChart(canvasId)` 辅助函数：检查 `STATE.charts[id]` 存在则 `.destroy()` 并 `delete`

**问题2**: 出勤页面"复位"按钮危险，用户要求移除。
**修改**: 删除第347行 `↺ 复位` 按钮，保留 `resetAttendance()` 函数和后端 `/attendance/reset` 接口（不影响其他调用）。
